Core
Validates resources at admission using CEL expressions embedded in YAML
Runs in-process within the API server — no external webhook, no network hop
ValidatingAdmissionPolicyBinding connects a policy to namespaces/resources
Parameterization via ParamKind — reuse policies with different ConfigMaps per env
MutatingAdmissionPolicy available in K8s 1.32+ (same model, for mutations)

Enforcement Modes
Deny — block the request from persisting
Warn — allow but surface a warning to the caller
Audit — record decision to audit log only
Modes can be combined: e.g. [Warn, Audit] simultaneously
validationActions field on Binding controls which modes are active

CEL Expressions
Familiar syntax similar to JavaScript/Go — easy for existing K8s devs
Pre-compiled: CEL programs compiled once, execution cost pre-calculated
Multiple validation expressions per policy, each with own error message
messageExpression — dynamic error messages using actual resource field values
has(), size(), matches(), startsWith() and many more built-in functions

Limitations to Know
Validation only — no mutation, generation, or image verification natively
No built-in PolicyReport CRDs — violations go to audit logs or metrics only
No background scanning of existing resources — admission-time only
CEL cannot call external services or perform API lookups
No CLI tooling for shift-left CI testing out of the box
Requires Kubernetes 1.30+ for stable VAP
